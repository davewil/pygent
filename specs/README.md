# Project Specifications & Documentation

This directory contains the core documentation, vision, and roadmap for the project.

## Development Practices

The following development practices are enforced to ensure code quality and maintainability:

### Code Standards
- **Typing**: Full static type hints are required (strict `mypy` compliance).
- **Docstrings**: Adhere to the **Google style** for docstrings.
- **Line Length**: Maximum line length of **120 characters**.
- **Formatting**: Use `ruff format` for code formatting.
- **Linting**: Use `ruff check` for linting.

### Testing
- All new features and bug fixes must be accompanied by relevant unit tests using `pytest`.
- Maintain >80% test coverage.
